System Requirements:

An IBM compatible PC with a minimum of 32 Mb of system memory, 4 Mb of free/available disk space, Windows 95/98, or Window NT Workstation 4.0. The PC operating system should have the most recent service packs installed. A modem or LAN connection to the Internet is required.

The web browsers to be used to access and update the agency database assets should be Internet Explorer 4.0 with Service Pack 1 installed or Netscape Navigator 4.0 Version 4.07. Older web browser versions will not work effectively.

User ID/Web Address:

Agencies will initially be allowed to have two (2) user id’s if they wish. One user id will be for the person updating the asset records, and the other user id will be for persons in the agency allowed to view asset records. Each agency that wishes to access the Y2K Web Database System will be required to e-mail John Oliver the first name, middle initial, and last name of the person that will be updating the records for their agency. The agency should also provide similar information for the user id that will view the records/assets of their agency. Please make sure to identify the user that will update the data and the user that will view the records. The user id format we will try to standardize on is the first letter of the user’s first name and the user’s last name concatenated together (Eg., Max Williams, user id will be MWilliams).

Once a request for a user id is made, we will return your e-mail with the user id’s, the initial password, and how to access the Y2K Web Database System. The URL for the Y2K database is as follows: http://y2k.state.ks.us

Sign On Screen:

Once you correctly access the Y2K Web Database, you must successfully sign onto the system to access the data.

  • Enter the user id provided by the staff of the Kansas Information Technology Office (KITO) and tab to the password field.
    •

 

  • Enter the initial password given to you, tab and hit the enter key, or click on sign on button.

If this is the first time you have signed onto the system with the password supplied by the KITO staff, you should receive a screen requesting you to provide a new password.

  • Type in a new password, hit the tab key or mouse click on the verify new password.

 

  • Re-key the password you just entered, tab and enter, or mouse click on the change button. I would recommend using a password that is at least seven (7) characters long.

 

Once you have successfully signed onto the system, the screen below will be displayed. The buttons across the top of the screen provide the functionality for the system.

The information to be entered/updated into the Y2K database conforms to the instructions provided for the Turn Around documents. The instructions and grading criteria are included at the back of this document.

The Administrative menu will be used only by the staff of the Kansas Information Technology Office (KITO) to administer the system. This area is used to add/remove users, update the Master Database (which will provide grading information for each Agency) from the Web Database, and other miscellaneous duties/functions.

 

Reports:

The reports option allows the Agency to view or print a hard copy of their information. To create a report, select the type of report from the list below.

List of Entered Assets Report:

If you selected the "List of Entered Assets" report, you will be prompted for some additional information listed below. Make your selections and click on the OK button.

List of Entered Assets Report continued:

The screen below contains the "List of Entered Assets" you selected from the previous screen. If you wish to obtain a printed (hard copy) of the report, hold down on the Ctrl key on the keyboard and press the P key. This should bring up a print dialog box that will allow you to select various options for printing your report to a printer.

Generate Agency Grade Report:

If you select the Generate Agency Grade Report, you will be viewing a report similar to the one below for your Agency. Again, to print a "hard copy" of the report, hold down on the Ctrl key and hit the P key. You should receive a printer dialog box to select various print options.

View Static Grade Report:

This option is currently inactive. The use for this report is still being evaluated.

Print Turnaround Documents/Reports:

This report basically provides the Agency a detail record of the database contents for each of their IT assets. Agencies are to update the IT Asset database by the 10th of each month so progress can be monitored and issues addressed in a timely manner. If an Agency does not show activity, they will be sent a copy of the turnaround document and asked to update the document with any CHANGES to their IT Assets. Again, a hard copy of the report can be obtained by holding the Ctrl key and hitting the P key. The user completes the information on the print dialog box and then the information will be sent to a network printer.

Print Turnaround Documents/Reports continued:

Adding New Records/Assets:

When an Agency needs to add assets to their listing, the user clicks of the Add Asset Records button and the following screen will present itself. The Agency you would like to add the asset to should already be present in the Agency box. The user should click on the OK button to proceed.

Adding New Records/Assets continued:

The screen below allows the Agency the ability to add the record to the database with the appropriate information for that asset.

Updating an Agency Asset:

The following screen displays when the Update Asset Records button is clicked on. The user selects the appropriate selection criteria or identifies the asset key (if you wish to obtain a particular record) and click on the search button.

Updating an Agency Asset continued:

The system will bring up a list of assets that met the search criteria selected on the prior screen. (Note: If you selected a specific asset key, this screen will be bypassed and the individual asset screen will appear - see the following page). To update a record, the user will click on the blue Asset Name and the individual asset screen will display (see the following page). An example of the Asset Name to click on for updating from below is "Admin Applications Arch Services".

 

Updating an Agency Asset continued:

The individual asset screen below will appear with the current information displayed. The user tabs and updates the appropriate fields. When the user has updated all the information on an asset, click on the Update button. If the user needs to update/add interface information, the user should click on the Update button before selecting/clicking on the interface information.

Agency Asset Update - Interfaces:

If the Agency had any interfaces already entered into the system, the screen below will list them; otherwise, the screen will display "No records" with a Add button so the Agency can add the new interfaces. If the Agency is updating an interface, click on the blue Exchanged With Agency (area next to the Add/Delete buttons below) and the screen on the following page will be displayed.

Agency Asset Update - Interfaces continued:

If the Agency is Adding or Updating an interface record, the following screen will be displayed. The Agency will need to update the record with the required information and click on the Add/Update button.

Deleting an Asset Record:

Any Agency that needs to remove/delete a record will need to send an e-mail or fax to the KITO staff with the following information:

  • Identify the Agency
  • Identify the Asset Key (run a Turnaround Report, at the right part of the report is the Asset Key. We will delete records ONLY if we are given the Asset Key.
  • Provide the Asset Name
  • Provide the reason the Asset(s) need to be removed.

The information can be faxed to (785) 296-1168 with Attn: KITO on the Fax or you can e-mail the information to us.

JohnO@dadisc1.wpo.state.ks.us

MaxW@dadisc1.wpo.state.ks.us

 

Printing a Hard Copy Report:

To obtain a printed report (hard copy), select the report you wish to print, when the report appears on the screen, hold down on the Ctrl key and hit the P key. A print dialog box will appear that will allow you to change the print characteristics as you require.

Reporting Requirements:

All Agencies are required to update the IT Asset database by the 10th of each month so progress can be monitored and issues addressed in a timely manner. If an Agency does not show activity, they will be sent a copy of the turnaround document and asked to update the document with any CHANGES to their IT Assets.

Updates can be done via the Web Database system described here, or CHANGES to their assets can be noted on the turnaround document and faxed or mailed to the KITO office by the 10th of the month. If you mail or fax your changes, please make sure the Agency name is on your documents, and your name and phone number are present.

TURN AROUND DOCUMENT INSTRUCTIONS

CURRENT REPORT INFORMATION

    1.    Review the document for completeness. Are all of your Y2K susceptible
          assets listed? Are there any assets listed in error?

    2.    Does the ACTION column accurately portray you mitigation activity? If not
           please enter the appropriate action. Possible actions include:

    Assessment Pending: The asset has been identified as having the potential for Y2K problems but is awaiting final assessment.

    Repair: The asset has been assessed, it will not function properly after the millennium, it will be repaired. On the form you will need to identify the Plan Start, Test, and Finish dates.

    Replace: The asset has been assessed, it will not function properly after the millennium, it will be replaced. NOTE: If a Custom software asset is being replaced please indicate what is replacing it, the anticipated Plan test date for the software to be Y2K ready, and provide the information that is requested for all assets.

    Discard: The asset has been assessed, it will not function properly after the millennium and it will be discarded. Please identify the final Plan date the application is to be discarded.

    No Action Required: The asset has been assessed and it will function properly after the millennium. Example: A Software asset that does not process or store dates, an imbedded processor that does not have a calendar function.

    3.   Does the STATUS column accurately reflect the assets mitigation activity
          status? If not please enter the appropriate status.

          Acceptable status for assets include:

      Repaired: The asset has been repaired and tested by the repairer. Example: code has been modified to include four digit years, system software has been replaced with a Y2K "ready" version, PC BIOS upgraded.

      User Tested & Accepted: The asset has been repaired or replaced and fully user tested to operate properly through the millennium. The user has signed off that the asset is ready.

      3rd Party Verified: The asset has been repaired or replaced and fully user tested to operate properly through the millennium. The user has signed off that the asset is ready. A third party signed reviewed repair and testing procedures and if necessary further tested the asset to assure it is ready.

      Active Repair: The asset is not Y2K ready but it is currently being repaired, or the replacement asset is being readied.

      Hold: The asset has been determined to require repair or replacement but activity has not been initiated.

      Vendor certified: The asset has been repaired or replaced and fully user tested to operate properly through the millennium. The user has signed off that the asset is ready. The asset is vendor certified to be Y2K ready.

      READY: This is NO longer a valid status.

    4.  If the asset is listed on your Year 2000 report and includes planned and
        actual start, test and finish dates along with estimated staff hours, staff hours
        to date, estimated vendor dollars and dollars to date, please indicate this
        on the form and attach the appropriate pages of your Y2K report. If the
        information isnot on your Y2K report please provide the information on the
        form.

NEW INFORMATION

5. Mission Critical (MC). Please code the application’s importance to State survival
    by rating this column according to the following criteria:

"1" Failure of the asset will result in wide public safety, health, welfare, or economic impact, or it will have serious liability consequences for the state and there is no viable "work around".

"2" Failure will result in a serious inability to perform normal government operations, but a "work around" is achievable for a limited period of time.

"3" Failure will have limited impact on the public or on government operations, or a viable "work around" is achievable for a sustained period of time.
  1. Custom Software Interfaces. There are 3 interface categories, Federal, Local and Other. Interfaces with other state agencies should be listed under Other. Please indicate if the data exchange is incoming (X) or outgoing (O) by placing an X or an O in the Quantity column. Please list the Agency or governmental unit with whom data is exchanged in the Action/Status columns. List the exchange medium under Planned and the agency/governmental unit contact under Actual. Room has been provided for up to three interfaces in each category, if more space is need please copy the page, mark it continuation and continue. NOTE: It is not necessary to list each and every county. Simply indicate all counties in the Action/Status column.

Agency Grading Guidelines:

The grading guidelines being used to determine your Y2K readiness is based on the information your included in these turnaround documents. The grading will be done using the following scale:

KANSAS MISSION CRITICAL APPLICATIONS/INTERFACES

Note: Once an asset received a status of User Tested & Accepted, Vendor Certified, or Third Party Verified, an "A" grade will be assigned to the asset.

Effective May 1, 1999, if the project has not completed within sixty (60) days of the Planned Finish Date, the asset will drop one (1) letter grade.

Effective July 1, 1999, if the project has not completed within thirty (30) days of the Planned Finish Date, the asset will drop one (1) letter grade.

By mid-May, 1999, we will provide to the Agency an Average Asset Grade Report. This grade will be their grade based on all their assets.

Also, by mid-May, 1999, we will be producing a grade report for Assets determined to be most critical to the support of essential Government services. CITO points will be displayed in this report. The grading scale will utilize the same scale as custom software assets.
